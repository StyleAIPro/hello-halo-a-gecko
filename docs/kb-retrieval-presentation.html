中英文混合分词

extractKeywords() 分词策略

英文处理

// 正则提取单词，过滤停用词 const words = text.match(/[a-z0-9]{2,}/g) .filter(w => !STOP_WORDS.has(w)) // 例: "UAV beamforming optimization" // → ["uav", "beamforming", "optimization"]

停用词过滤

theisarehave waswillcanbut notthisthatwith forfromandor
的了是在不是和也 与有也可还是也有

中文处理

// 短段（≤4字）：整体作为一个词 "波束赋形" → ["波束赋形"] // 长段：bigram + trigram "波束赋形优化方案" bigrams: 波束赋、束赋形、赋形优、形优化、化方案 trigrams: 波束赋、束赋形、赋形优、形优化 // 去重：被 trigram 覆盖的 bigram 移除

设计理由：中文无空格分词，采用 N-gram 策略避免引入外部分词依赖。Trigram 捕获更长的语义单元，Bigram 补充未被覆盖的短词。

字段权重

标题和标签权重是正文的 3 倍，确保主题匹配优先于内容提及

评分示例

Query: "UAV 波束赋形优化"

Keywords: ["uav", "波束赋", "束赋形", "赋形优", "形优化"] Page "Hybrid Beamforming for UAV" title: "uav" +1.5, "beamforming" +1.5 body: "uav" +0.5 tag: "uav" +1.5 Score = 5.0 ✓ Page "Analog Beamforming" title: "beamforming" +1.5 body: "beamforming" +0.5 Score = 2.0

索引补充：还会匹配 index.md（知识库目录页）中的条目行，命中时额外加 score × 0.8

08 / 10

两跳关联扩展

Hop 2: 通过 Wikilink 发现间接相关页面

Wikilink

[[页面标题]] 语法

Wiki 页面间通过 [[wikilink]] 建立双向引用关系。摄取完成后自动交叉链接（crossLinkAllPages）。

算法逻辑

  • 取 Hop1 得分 ≥ 1 的前 8 个页面
  • 读取这 8 个页面的正文，提取所有 [[link]]
  • 链接目标页面获得 0.5 分基础分
  • 支持模糊标题匹配（大小写、别名变体）
  • 已访问链接去重，避免重复加分

关联发现示例

Page A: Hybrid Beamforming
Hop1 Score: 5.0
包含 [[Constant-Modulus Constraint]] [[SDR]]
Page B: CM Constraint
+0.5
Page C: SDR

价值：即使用户查询关键词未出现在 Page B/C 中，它们因与高分页面 A 存在 Wikilink 关联而被召回。这模拟了"相关主题推荐"的效果。

设计总结

核心特性一览

特性说明优势
检索方式 倒排索引 + 关键词匹配 无向量依赖，零外部服务调用
分词策略 英文单词 + 中文 bigram/trigram 纯本地实现，支持中英文混合
关联扩展 Wikilink 两跳扩散 发现间接相关页面，提高召回率
上下文管理 10K Token 截断 控制成本，保证响应速度
引用溯源 [N] 标记 + 文末参考文献 可验证、可跳转的知识来源
会话隔离 Per-Conversation KB 选择 不同对话使用不同知识库组合

纯本地 · 零外部依赖 · 毫秒级检索 · 完整引用溯源
