# 02 - 主进程模块设计

## 1. Agent 系统

Agent 系统是 AICO-Bot 的核心引擎，负责与 Claude Agent SDK 交互，管理对话会话、流式响应、工具调用权限等。

### 1.1 模块组成

```
src/main/services/agent/
├── orchestrator.ts        # 多智能体编排器 (单例)
├── session-manager.ts     # 会话生命周期管理
├── stream-processor.ts    # 流式响应处理管线
├── send-message.ts        # 消息发送统一入口
├── sdk-config.ts          # SDK 配置构建
├── sdk-turn-injection-patch.ts  # SDK 运行时补丁
├── control.ts             # 生成控制 (停止/状态)
├── helpers.ts             # 工具函数
├── message-utils.ts       # 消息解析工具
├── permission-handler.ts  # 工具权限处理
├── permission-forwarder.ts# Worker 权限转发
├── hyper-space-mcp.ts     # Hyper Space MCP 服务器
├── mailbox.ts             # 文件邮箱服务
├── taskboard.ts           # 任务看板服务
├── persistent-worker.ts   # 持久化 Worker 循环
├── system-prompt.ts       # 系统提示词模板
├── mcp-manager.ts         # MCP 状态管理
├── types.ts               # 核心类型定义
└── index.ts               # 公共导出
```

### 1.3 消息发送入口 (`send-message.ts`)

`sendMessage(request: AgentRequest)` 是 Agent 系统的统一入口，根据空间配置选择三条执行路径之一：

```
sendMessage()
    │
    ├──► 路径1: 远程代理 (space.claudeSource === 'remote')
    │     executeRemoteMessage()
    │     ├── SSH 隧道建立 (可选)
    │     ├── WebSocket 连接远程代理
    │     ├── MCP Bridge 暴露本地工具
    │     └── 流式响应回传
    │
    ├──► 路径2: Hyper Space 多智能体
    │     agentOrchestrator.executeOnSingleAgent()
    │     ├── Leader Agent 执行
    │     ├── spawn_subagent 创建 Worker
    │     ├── Worker 公告注入 Leader 会话
    │     └── 最大 20 轮注入循环
    │
    └──► 路径3: 本地标准执行
          executeLocalMessage()
          ├── getOrCreateV2Session()
          ├── processStream()
          └── SDK stream() 流式处理
```

### 1.4 编排器 (`orchestrator.ts`)

`AgentOrchestrator` 是单例类，继承 `EventEmitter`，负责多智能体团队管理。

#### 核心概念

- **AgentInstance**: 单个智能体实例，包含 id、name、role、capabilities、persistence mode
- **AgentTeam**: 智能体团队，包含 Leader + Workers + Mailbox + TaskBoard
- **Leader Agent**: 负责任务分发、结果汇总的主智能体
- **Worker Agent**: 执行具体任务的从智能体

#### 核心方法

| 方法 | 说明 |
|------|------|
| `createTeam(params)` | 创建智能体团队，初始化邮箱和任务看板 |
| `executeOnSingleAgent(params)` | 单智能体执行入口 |
| `executeAgentLocally(agent, task, conversationId, ...)` | 本地执行，使用 V2 会话 + 注入循环 |
| `executeAgentRemotely(...)` | 远程执行，WebSocket + SSH 隧道 |
| `dispatchTask(params)` | 任务分发: 能力匹配、轮询、手动指派、广播 |
| `executeAllTasks(teamId)` | 并行或顺序执行所有待处理任务 |
| `sendAnnouncement(announcement)` | 将 Worker 完成公告注入 Leader 会话 |
| `waitForCompletion(params)` | 轮询等待公告完成，心跳超时延长 |
| `destroyTeam(teamId)` | 级联清理: Workers、Tasks、Mailboxes、Injections |
| `startPersistentWorkers(teamId)` | 启动持久化 Worker 循环 |

#### 关键常量

- 最大注入循环次数: **20**
- 最大衍生深度: **50**
- 每个 Agent 最大子进程数: **5**
- 子对话 ID 模式: `${parentConversationId}:agent-${agentId}`

### 1.5 会话管理器 (`session-manager.ts`)

模块级管理的会话生命周期服务。

#### 核心状态

```typescript
activeSessions: Map<string, SessionState>    // 进行中的生成会话
v2Sessions: Map<string, V2SessionInfo>        // V2 持久会话池 (按 conversationId 键)
pendingInvalidations: Set<string>             // 延迟会话重建
sessionHealthMap: Map<string, SessionHealth>  // 会话健康度追踪
```

#### 核心函数

| 函数 | 说明 |
|------|------|
| `getOrCreateV2Session(spaceId, conversationId, sdkOptions, ...)` | 核心: 会话创建/复用，检查传输就绪、检测配置重建、追踪凭证代数 |
| `ensureSessionWarm(spaceId, conversationId)` | 预热会话，加速首次响应 |
| `closeV2Session(conversationId)` | 清理会话 |
| `closeSessionsBySpaceId(spaceId)` | 批量关闭 (空间删除时) |
| `invalidateAllSessions()` | 延迟重建 (如有请求进行中) 或立即关闭 |
| `compactContext(conversationId)` | 手动调用 SDK compact() 压缩上下文 |
| `isSessionTransportReady(session)` | 检查 SDK 内部 transport.ready 状态 |
| `migrateSessionIfNeeded(workDir, sessionId)` | 迁移旧会话文件到新目录 |

#### 会话健康监控

- 健康检查间隔: **60 秒**
- 最大连续失败次数: **5** 次后重启
- 请求超时: **45 分钟**
- 会话空闲超时: **30 分钟**

### 1.6 流处理器 (`stream-processor.ts`)

核心函数 `processStream(params: ProcessStreamParams)` 处理 SDK 流式响应。

#### ProcessStreamParams 接口

```typescript
interface ProcessStreamParams {
  v2Session: V2SDKSession
  sessionState: SessionState
  spaceId: string
  conversationId: string
  messageContent: string
  displayModel: string
  abortController: AbortController
  t0: number
  callbacks: StreamCallbacks
  rendererConversationId?: string
  suppressComplete?: boolean
  workerInfo?: WorkerInfo
  contextWindow?: number
}

interface StreamCallbacks {
  onComplete(result: StreamResult): void
  onRawMessage?(sdkMessage: SDKMessage): void
}

interface StreamResult {
  finalContent: string | null
  thoughts: Thought[]
  tokenUsage: TokenUsage
  capturedSessionId: string
  isInterrupted: boolean
  wasAborted: boolean
  hasErrorThought: boolean
  errorThought: Thought | null
  reachedMaxTurns: boolean
  hasPendingInjection: boolean
  needsAuthRetry: boolean
}
```

#### SDK 事件类型处理

| SDK 事件 | 子类型 | 处理方式 |
|----------|--------|----------|
| `content_block_start` | thinking, text, tool_use | 创建新的 Thought 块 |
| `content_block_delta` | thinking_delta, text_delta, input_json_delta | 追加到当前 Thought 内容 |
| `content_block_stop` | - | 标记 Thought 完成 (isStreaming = false) |
| 非流消息 | system, assistant, user, result | 直接处理 |

#### 渲染进程事件发送

流处理器通过 IPC 向渲染进程发送以下事件：

| 事件通道 | 说明 |
|----------|------|
| `agent:thought` | 新思考块 |
| `agent:thought-delta` | 思考增量 |
| `agent:message` | 文本消息 |
| `agent:tool-call` | 工具调用 |
| `agent:tool-result` | 工具结果 |
| `agent:turn-boundary` | 轮次边界 |
| `agent:complete` | 生成完成 |
| `agent:error` | 错误 |
| `agent:compact` | 上下文压缩 |
| `worker:started` | Worker 启动 |
| `worker:completed` | Worker 完成 |

#### 注入系统

```typescript
queueInjection(message: string, conversationId: string)    // 排队注入
getAndClearInjection(conversationId): string | null          // 取出并清除
hasPendingInjection(conversationId): boolean                 // 检查是否有待注入
clearInjectionsForConversation(conversationId)               // 清除指定注入
clearAllInjections()                                         // 清除所有注入
```

最大注入循环次数: **20**

### 1.7 SDK 配置 (`sdk-config.ts`)

#### 凭证解析

```typescript
resolveCredentialsForSdk(credentials): ResolvedCredentials
```

- Anthropic 提供商: 当 `PROXY_ANTHROPIC=true` 时路由到本地代理
- OpenAI 配置: 编码到 API Key 中传递

#### SDK 选项构建

```typescript
buildBaseSdkOptions(params): SDKOptions
```

构建包含环境变量、系统提示词、MCP 服务器、权限处理器的完整 SDK 选项。

```typescript
{
  model: 'claude-sonnet-4-20250514',
  cwd: workDir,
  permissionMode: 'bypassPermissions',
  includePartialMessages: true,
  maxThinkingTokens: 10240  // 思考功能启用时
}
```

#### 环境变量清理

`getCleanUserEnv()`: 剥离 `ANTHROPIC_`、`OPENAI_`、`CLAUDE_` 前缀变量，防止冲突。

配置目录: `~/.agents/claude-config/` (通过 `CLAUDE_CONFIG_DIR` 环境变量指定)

### 1.8 SDK 运行时补丁 (`sdk-turn-injection-patch.ts`)

对 SDK 二进制进行运行时补丁，支持对话中间注入用户消息。

#### 补丁内容

- 在 Query 类添加 `_continueAfterResult` 和 `_pendingUserMessages` 属性
- 修改 `send()` 方法: 队列化消息
- 修改 `readMessages()`: 在结果后注入用户消息
- 修改 `stream()`: 继续迭代
- 导入时自动补丁

### 1.9 权限处理 (`permission-handler.ts`)

`createCanUseTool()` 工厂函数:

- 自动允许所有工具调用 (permissionMode: bypassPermissions)
- 遇到 `AskUserQuestion` 工具时暂停，通过 IPC 等待用户输入
- 用户响应后恢复执行

### 1.10 系统提示词 (`system-prompt.ts`)

系统提示词模板，支持占位符替换。构建时考虑以下因素：

- 工作目录
- 可用工具
- 上下文信息
- 团队角色信息 (多智能体模式)

## 5. 持久化 Worker (`persistent-worker.ts`)

`PersistentWorkerLoop` - Worker 智能体的持续轮询循环。

#### 常量

- 轮询间隔: **3000ms**
- 最大空闲等待: **30000ms**
- 关闭超时: **30000ms**

#### 主循环

```
while (running) {
  1. 轮询邮箱消息
  2. 自动认领任务
  3. 空闲等待
}
```

#### 消息类型

- `chat/direct` - 聊天消息
- `task_assignment` - 任务分配
- `shutdown_request` - 关闭请求
- `permission_response` - 权限响应
