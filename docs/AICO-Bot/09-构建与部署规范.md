# 09 - 构建与部署规范

## 1. 构建系统

### 1.1 技术栈

| 工具 | 版本/说明 |
|------|-----------|
| electron-vite | 构建打包工具，支持 main/preload/renderer 多入口 |
| electron-builder | 分发打包，支持 macOS/Windows/Linux |
| Vite | 渲染进程开发服务器和构建 |
| TypeScript | 类型系统，项目引用模式 (project references) |
| React 18 | 渲染进程 UI 框架 |

### 1.2 构建配置 (`electron.vite.config.ts`)

#### 三个构建目标

| 目标 | 入口 | 输出 |
|------|------|------|
| **main** | `src/main/index.ts` | `out/main/` |
| **preload** | `src/preload/index.ts` | `out/preload/` |
| **renderer** | `src/renderer/main.tsx` | `out/renderer/` |

#### Worker 构建

- 入口: `src/worker/file-watcher/index.ts`
- 输出: 打包到 main 构建中

#### HTML 入口

- `index.html` - 主窗口页面
- `overlay.html` - 覆盖窗口页面

#### 路径别名

```typescript
resolve: {
  alias: {
    '@': path.resolve('src/renderer'),
  }
}
```

#### 构建时注入

通过 `define` 注入 Analytics 配置等编译时常量。

## 2. 开发命令

### 2.1 开发服务器

```bash
npm run dev              # 启动开发服务器
                         # 使用 ~/.aico-bot-dev 数据目录
                         # Vite 开发服务器端口: 5173
```

### 2.2 构建命令

```bash
npm run build            # 构建 proxy + electron-vite build (输出到 out/)
npm run build:mac        # 构建 macOS 通用版本 (dmg + zip)
npm run build:win        # 构建 Windows x64 (nsis 安装包)
npm run build:linux      # 构建 Linux x64 (AppImage)
```

### 2.3 测试命令

```bash
npm run test                     # 二进制检查 + 单元测试
npm run test:unit                # vitest 运行所有单元测试
npm run test:unit:watch          # vitest watch 模式

npm run test:e2e                 # Playwright E2E (需先 npm run build)
npm run test:e2e:smoke           # E2E 冒烟测试
npm run test:e2e:headed          # E2E 带浏览器界面

# 运行单个单元测试
npx vitest run --config tests/vitest.config.ts \
  tests/unit/services/config.test.ts

# 按模式匹配运行测试
npx vitest run --config tests/vitest.config.ts \
  -t "should return default config"
```

### 2.4 国际化命令

```bash
npm run i18n             # 提取 + 翻译 (需要 .env.local 中的 AICO_BOT_TEST_* 变量)
npm run i18n:extract     # 仅提取键
npm run i18n:translate   # 仅 AI 翻译
```

### 2.5 二进制准备

```bash
npm run prepare          # 下载当前平台的二进制文件
npm run prepare:all      # 下载所有平台的二进制文件
```

### 2.6 其他命令

```bash
npm run after-pack       # 打包后钩子
```

## 3. TypeScript 配置

### 3.1 配置文件

| 文件 | 用途 |
|------|------|
| `tsconfig.json` | 根配置，引用 node 和 web 配置 |
| `tsconfig.node.json` | 主进程/preload/shared/worker 配置 |
| `tsconfig.web.json` | 渲染进程配置 (含 JSX) |

### 3.2 tsconfig.node.json

```jsonc
{
  "compilerOptions": {
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "types": ["electron-vite/node"]
  }
}
```

### 3.3 tsconfig.web.json

```jsonc
{
  "compilerOptions": {
    "module": "ESNext",
    "jsx": "react-jsx",
    "lib": ["DOM", "ESNext"]
  }
}
```

### 3.4 路径别名

```jsonc
{
  "paths": {
    "@/*": ["src/renderer/*"],
    "@main/*": ["src/main/*"],
    "@shared": ["src/shared/*"],
    "@shared/*": ["src/shared/*"]
  }
}
```

## 9. 跨平台注意事项

### 9.1 路径处理

- Windows 使用反斜杠 `\`，macOS/Linux 使用正斜杠 `/`
- 使用 `path.join()` 和 `path.resolve()` 处理路径
- 数据目录: `~/.aico-bot/` (跨平台使用 `os.homedir()`)

### 9.2 端口管理

| 平台 | 端口检查 | 进程终止 |
|------|----------|----------|
| Windows | `netstat -ano \| findstr` | `taskkill /F /PID` |
| macOS/Linux | `lsof -i :port -t` | `kill` |

### 9.3 GPU 兼容性

Windows 下默认禁用硬件加速 (`app.disableHardwareAcceleration()`)，防止空白窗口问题。

### 9.4 托盘图标

- macOS: 使用 `trayTemplate.png` (系统模板图像，自动适配深色/浅色模式)
- Windows: 使用 `tray-16.png` (固定尺寸)

```bash

```

开发服务器特性:
- 数据目录: `~/.aico-bot-dev/`
- Vite 端口: 5173

### 10.2 性能监控

渲染进程性能监控 (仅开发模式):
- `perf:start` / `perf:stop` IPC 通道
- `perf:snapshot` / `perf:warning` 事件

- 使用 `electron-log` 库
- 日志级别: error, warn, info, verbose, debug, silly
