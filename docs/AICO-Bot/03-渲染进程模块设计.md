# 03-渲染进程模块设计

## 3.1 概述

渲染进程是 AICO-Bot 的用户界面层，基于 React 18 + TypeScript 构建，运行在 Electron 的 WebContents 中。渲染进程通过 IPC（Electron 模式）或 HTTP + WebSocket（远程 Web 模式）与主进程通信，为用户提供 AI Agent 交互、文件浏览、自动化应用管理等能力。

**入口文件：**
- `src/renderer/main.tsx` — 主应用入口
- `src/renderer/overlay-main.tsx` — 叠加层应用入口（独立 SPA）

**技术栈：**
- React 18（函数组件 + Hooks）
- Zustand（前端状态管理）
- Tailwind CSS（CSS 变量主题系统）
- i18next（国际化，支持 7 种语言）
- Vite（构建工具）
- electron-log（Electron 环境日志）

## 3.2 React UI 架构

### 3.2.1 应用入口与初始化

**文件：** `src/renderer/main.tsx`

```
main.tsx
  |-- electron-log 初始化（仅 Electron 环境）
  |-- i18n 配置导入
  |-- CSS 层叠导入（globals.css > syntax-theme.css > canvas-tabs.css > browser-task-card.css）
  |-- ErrorBoundary 包裹
      └── App.tsx
```

入口文件中通过检测 `window.aicoBot` 判断是否在 Electron 环境中，仅在 Electron 环境下初始化 `electron-log`。设置了全局标记 `__AICO_BOT_APP_MOUNTED__`，用于禁用 `index.html` 中的全局错误降级处理（由 React ErrorBoundary 接管）。

### 3.2.2 App 组件架构

**文件：** `src/renderer/App.tsx`

`App.tsx` 是整个渲染进程的根组件，承担以下核心职责：

#### （1）启动初始化（Pull+Push 模式）

采用 Pull+Push+Timeout 三重保障机制确保后端服务就绪后再初始化 UI：

```
Pull（拉取）：api.getBootstrapStatus() → 立即查询状态（处理 HMR 重载、错误恢复）
Push（推送）：api.onBootstrapExtendedReady() → 监听主进程事件（正常启动流程）
Timeout（超时）：5000ms 后备保护 → 强制初始化
```

#### （2）视图路由系统

App 不使用第三方路由库，而是基于 `useAppStore().view` 状态驱动视图切换。支持的视图类型：

| 视图 | 组件 | 说明 |
|------|------|------|
| `splash` | `SplashScreen` | 启动画面 |
| `gitBashSetup` | `GitBashSetup` | Windows Git Bash 安装向导 |
| `setup` | `SetupFlow` | 首次设置引导 |
| `home` | `HomePage`（懒加载） | 主页 |
| `space` | `SpacePage`（懒加载） | 工作空间页 |
| `settings` | `SettingsPage`（懒加载） | 设置页 |
| `apps` | `AppsPage`（懒加载） | 自动化应用管理页 |
| `skill` | `SkillPage`（懒加载） | 技能管理页 |
| `remoteServers` | `RemoteServersPage`（懒加载） | 远程服务器管理 |
| `remoteChat` | `RemoteAgentChatPage`（懒加载） | 远程 Agent 聊天 |

复杂页面组件均使用 `React.lazy()` + `Suspense` 进行代码分割，配合 `PageLoader` 降级 UI，优化首屏加载性能。

#### （3）全局事件监听注册

`App.tsx` 在 `useEffect` 中统一注册所有 Agent 事件监听器，这些监听器是全局的（不绑定到特定会话）：

- **Agent 核心事件：** `onAgentMessage`、`onAgentToolCall`、`onAgentToolResult`、`onAgentError`、`onAgentComplete`
- **思考过程事件：** `onAgentThought`、`onAgentThoughtDelta`
- **上下文管理：** `onAgentCompact`（压缩通知）
- **用户交互：** `onAgentAskQuestion`、`onAgentTurnBoundary`（回合边界注入）
- **多 Agent 协作：** `onAgentTeamMessage`、`onWorkerStarted`、`onWorkerCompleted`
- **应用系统事件：** `onAppStatusChanged`、`onAppActivityEntry`、`onAppEscalation`、`onAppNavigate`
- **通知系统：** `onNotificationToast`

#### （4）主题切换

```tsx
applyTheme(theme: 'light' | 'dark' | 'system')
  |-- 写入 localStorage（防闪烁）
  |-- 切换 <html> 的 .light 类
  |-- system 模式：监听系统主题变化
  |-- 调用 api.setTitleBarOverlay() 同步标题栏颜色（Windows/Linux）
```

### 3.3.2 chat.store.ts 核心架构

`chat.store.ts` 是渲染进程最核心的 Store（3257 行），管理所有与对话相关的状态。

#### （1）三层数据模型

```
┌─────────────────────────────────────────────────────────┐
│  SpaceState (per space)                                  │
│  spaceStates: Map<spaceId, SpaceState>                  │
│    └── conversations: ConversationMeta[]  ← 轻量元数据    │
│    └── workerConversations: Map ← Worker 会话元数据       │
├─────────────────────────────────────────────────────────┤
│  ConversationCache (on-demand)                           │
│  conversationCache: Map<conversationId, Conversation>   │
│    └── LRU 缓存，上限 10 个完整对话                       │
├─────────────────────────────────────────────────────────┤
│  SessionState (per conversation, runtime)               │
│  sessions: Map<conversationId, SessionState>            │
│    └── 运行时状态，跨 Space 切换保持                      │
└─────────────────────────────────────────────────────────┘
```

#### （2）SessionState 结构

每个会话的运行时状态包含：

```typescript
interface SessionState {
  isGenerating: boolean           // Agent 是否正在生成
  isStopping: boolean             // 用户点击了停止，等待清理
  streamingContent: string        // 流式文本内容
  isStreaming: boolean            // 文本流式传输中
  thoughts: Thought[]             // 思考过程块列表
  isThinking: boolean             // 正在思考
  pendingToolApproval: ToolCall   // 待审批的工具调用
  error: string | null            // 错误信息
  errorType: AgentErrorType | null
  compactInfo: CompactInfo | null // 上下文压缩通知
  textBlockVersion: number        // 文本块版本号（用于 StreamingBubble 重置）
  pendingQuestion: PendingQuestion | null  // AskUserQuestion 待答问题
  pendingMessages: PendingMessage[]        // 排队等待的消息
  workerSessions: Map<string, WorkerSessionState>  // Hyper Space Worker 会话
}
```

#### （3）关键事件处理流程

**Agent 消息处理（handleAgentMessage）：**

```
主进程发送 agent:message 事件
  └── App.tsx onAgentMessage 回调
      └── chatStore.handleAgentMessage()
          ├── 解析 conversationId
          ├── 更新 streamingContent + isStreaming
          ├── 检查是否为 Worker 消息（含 :agent-* 后缀）
          ├── 触发 canvas 内容检测（scanCanvasForNewContent）
          └── 调用 applyWorkerStreamUpdate()（节流刷新）
```

**思考过程处理（handleAgentThought / handleAgentThoughtDelta）：**

```
agent:thought 事件（完整块）
  └── 创建 Thought 对象，添加到 thoughts 数组

agent:thought-delta 事件（增量更新）
  └── 根据 thoughtId 找到对应 Thought，追加 delta 内容
      └── 支持 toolInput/toolResult 增量
```

**Worker 会话状态管理：**

Hyper Space 支持多 Agent 协作，每个 Worker Agent 有独立的会话状态：

```typescript
interface WorkerSessionState {
  agentId: string
  agentName: string
  taskId: string | null
  task: string
  isRunning: boolean
  status: 'running' | 'completed' | 'failed'
  streamingContent: string
  isStreaming: boolean
  thoughts: Thought[]
  isThinking: boolean
  pendingQuestion: PendingQuestion | null
  interactionMode: 'mention' | 'delegation'  // @提及 vs 委派
  childConversationId?: string               // 子会话 ID（持久化历史）
  turnStartedAt: number                      // 当前轮次开始时间
}
```

Worker 流式更新采用节流机制（`workerStreamThrottleMap`），避免每收到一个 token delta 就触发 Zustand `set()` 导致性能问题。

#### （4）跨 Store 依赖

- `chat.store.ts` 依赖 `useTerminalStore`（终端状态）
- `chat.store.ts` 依赖 `useSpaceStore`（工作空间状态）
- `chat.store.ts` 依赖 `canvas-lifecycle` 服务（Canvas 内容扫描）

## 3.4 双模式 API 层

渲染进程通过统一的 `api` 对象与后端通信，自动适配 Electron IPC 和远程 HTTP 两种模式。

### 3.4.1 传输层（transport.ts）

**文件：** `src/renderer/api/transport.ts`

#### 环境检测

```typescript
isElectron()  → 检查 window.aicoBot 是否存在
isRemoteClient() → !isElectron()
```

#### IPC 事件映射（methodMap）


`onEvent(channel, callback)` 函数根据运行环境自动选择：
- **Electron 模式：** 调用 `window.aicoBot[method](callback)`
- **远程模式：** 通过 WebSocket 事件分发

#### WebSocket 连接管理（远程模式）

```
connectWebSocket()
  |-- 使用 token 建立 WebSocket 连接
  |-- 发送 auth 消息认证
  |-- 支持 subscribe/unsubscribe 会话订阅
  |-- 自动重连（3 秒间隔）
  |-- subscribe 确认机制（pendingSubscribeAcks，3 秒超时）
```

#### HTTP 请求封装

```typescript
httpRequest(method, path, body?)
  |-- 自动附加 Bearer Token
  |-- 401 自动清除 token 并刷新页面
  |-- 统一错误处理
```

#### 新增 IPC 通道的三步法

当需要添加新的 IPC 通道时，必须同步修改 3 个文件：

1. **`src/preload/index.ts`** — 通过 `window.aicoBot` 暴露方法
2. **`src/renderer/api/transport.ts`** — 在 `methodMap` 中添加事件映射
3. **`src/renderer/api/index.ts`** — 在 `api` 对象上导出方法

## 3.7 国际化（i18n）

### 3.7.1 配置

**文件：** `src/renderer/i18n/index.ts`

使用 `i18next` + `react-i18next` 方案，支持 7 种语言：

| 语言代码 | 显示名称 | 文件名 |
|---------|---------|--------|
| `en` | English | `en.json` |
| `zh-CN` | 简体中文 | `zh-CN.json` |
| `zh-TW` | 繁体中文 | `zh-TW.json` |
| `ja` | 日本語 | `ja.json` |
| `es` | Español | `es.json` |
| `fr` | Français | `fr.json` |
| `de` | Deutsch | `de.json` |

**源语言为英文** —— `t('English text')` 中的英文字符串本身就是英文翻译值。

### 3.7.2 系统语言检测

```
detectLanguage()
  |-- 精确匹配 navigator.language
  |-- 中文变体：zh-TW/zh-HK/zh-Hant → zh-TW，其他 zh → zh-CN
  |-- 其他语言按前缀匹配
  |-- 默认回退到 en
```

语言偏好存储在 `localStorage` 的 `aico-bot-locale` 键中。

### 3.7.3 使用方法

```tsx
import { useTranslation } from '../i18n'

function MyComponent() {
  const { t } = useTranslation()
  return <Button>{t('Save')}</Button>
}
```

### 3.7.4 添加新字符串流程

1. 在组件中直接使用 `t('新的英文文本')`
2. 运行 `npm run i18n:extract` 提取新键到所有 locale 文件
3. 运行 `npm run i18n:translate` 使用 AI 自动翻译（需配置 `AICO_BOT_TEST_*` 环境变量）
4. 或直接手动编辑各 `locales/*.json` 文件

**重要规则：** UI 中禁止出现硬编码的用户可见文本字符串。所有用户可见文字必须使用 `t()` 包裹。

## 3.8 叠加层窗口系统（Overlay Window）

### 3.8.1 架构设计

叠加层是一个**独立的 SPA 应用**，运行在独立的 `WebContentsView` 中，渲染在主 UI 和 BrowserView 之上。

```
主窗口 (BrowserWindow)
  └── contentView (根容器)
      ├── 主 UI WebContentsView（聊天、Canvas 等）
      ├── BrowserView(s)（AI 浏览器标签页）
      └── Overlay WebContentsView（叠加层 SPA — 最顶层）
```

**入口文件：** `src/renderer/overlay-main.tsx`

### 3.8.2 叠加层组件

**路径：** `src/renderer/overlay/`

```
overlay/
  |-- OverlayApp.tsx        # 叠加层根组件
  |-- ChatCapsuleOverlay.tsx # 聊天胶囊按钮（Canvas 最大化时显示）
  |-- overlay.css           # 叠加层专用样式（透明背景）
```

### 3.8.3 通信机制

```
主进程发送 overlay:state-change IPC 事件
  └── OverlayApp 接收并更新 state
      └── 控制 ChatCapsule 等浮层元素的显示/隐藏

用户点击胶囊按钮
  └── 发送 overlay:exit-maximized IPC 到主进程
      └── 主进程退出 Canvas 最大化模式
```

### 3.8.4 设计要点

- 叠加层**不导入** `globals.css`（会设置 body 背景色，破坏透明效果）
- 使用独立的 `overlay.css`，确保背景透明
- 共享 i18n 配置以保证多语言一致性
- 支持未来扩展：对话框、菜单、工具提示等浮层元素

## 3.9 Tailwind CSS 主题系统

### 3.9.1 CSS 变量主题

**文件：** `src/renderer/assets/styles/globals.css`

采用 CSS 自定义属性 + HSL 颜色格式实现主题系统：

```css
:root {
  /* 暗色主题（默认） */
  --background: 0 0% 4%;
  --foreground: 0 0% 98%;
  --card: 0 0% 7%;
  --primary: 217 91% 60%;        /* 品牌蓝 */
  --secondary: 0 0% 15%;
  --muted: 0 0% 15%;
  --border: 0 0% 20%;
  --radius: 0.75rem;

  /* AICO-Bot 品牌色 */
  --aico-bot-glow: 217 91% 60%;
  --aico-bot-success: 142 71% 45%;
  --aico-bot-warning: 38 92% 50%;
  --aico-bot-error: 0 62% 50%;
}

.light {
  /* 亮色主题 — 覆盖相同变量 */
  --background: 0 0% 100%;
  --foreground: 0 0% 9%;
  /* ... */
}
```

### 3.9.2 使用方式

组件中使用 Tailwind 的 CSS 变量引用：

```tsx
// 正确：使用 CSS 变量
<div className="bg-background text-foreground border-border">

// 错误：硬编码颜色值（禁止）
<div className="bg-white text-black border-gray-200">
```

### 3.9.3 样式文件结构

| 文件 | 职责 |
|------|------|
| `globals.css` | Tailwind 指令 + CSS 变量 + 基础样式 + 全局动画 |
| `syntax-theme.css` | highlight.js 代码语法高亮 |
| `canvas-tabs.css` | VS Code 风格标签栏样式 |
| `browser-task-card.css` | AI 浏览器科幻风格效果 |
| `overlay/overlay.css` | 叠加层专用样式（透明背景） |

### 3.9.4 样式隔离原则

`globals.css` 仅包含：
- Tailwind 指令（`@tailwind base/components/utilities`）
- CSS 变量定义
- `html`/`body` 基础样式
- 滚动条样式
- 共享 `@keyframes` 动画

组件特有样式放在独立文件中，避免全局污染。

## 3.10 数据流总结

### 3.10.1 Agent 消息流

```
Claude Code SDK (主进程)
  └── stream_event / message 事件
      └── IPC: sendToRenderer('agent:message', spaceId, conversationId, data)
          └── transport.ts: onEvent('agent:message', callback)
              └── App.tsx: onAgentMessage 回调
                  └── chatStore.handleAgentMessage()
                      └── 更新 sessions Map 中的 SessionState
                          └── React 组件自动重渲染（MessageList 等）
```

### 3.10.2 用户发送消息流

```
用户输入 → InputArea 组件
  └── chatStore.sendMessage()
      └── api.sendMessage(request)
          ├── Electron 模式: window.aicoBot.sendMessage()
          └── 远程模式: subscribeToConversation() → httpRequest('POST', '/api/agent/message')
              └── 主进程接收 → 启动 Claude Code SDK 执行
```

### 3.10.3 远程模式 WebSocket 事件流

```
远程服务器
  └── WebSocket 推送 event 消息
      └── transport.ts: wsConnection.onmessage
          └── 根据 message.channel 分发到 wsEventListeners
              └── 注册的回调函数处理
```
