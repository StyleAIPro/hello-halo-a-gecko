# 08 - 数据模型与存储设计

## 1. 存储架构概述

AICO-Bot 使用 **SQLite + 文件系统** 混合存储架构：

- **SQLite 数据库**: 结构化数据 (应用安装记录、自动化运行日志、调度任务、迁移版本)
- **文件系统**: 对话历史、Artifact 文件、应用工作目录、Memory 文件、邮箱消息、会话 JSONL

### 1.1 数据目录布局

```
~/.aico-bot/                          # 生产模式数据目录
~/.aico-bot-dev/                      # 开发模式数据目录
    │
    ├── aico-bot.db                    # 主 SQLite 数据库
    ├── spaces/                        # 工作空间数据
    │   └── {spaceId}/
    │       ├── mailboxes/             # 邮箱消息 (Hyper Space)
    │       │   └── {mailboxId}.jsonl
    │       ├── taskboard.json         # 任务看板
    │       └── ...
    │
    ├── memory/                        # 用户级内存
    │   ├── memory.md                  # 当前内存文件
    │   └── user-memory/               # 用户内存归档
    │
    └── apps/                          # 应用注册表缓存
        └── store/
```

```
{spacePath}/.aico-bot/                # 空间级数据目录
    │
    ├── conversations/                 # 对话持久化
    │   └── {conversationId}.jsonl
    │
    ├── apps/                          # 已安装应用
    │   └── {appId}/
    │       ├── memory.md              # 应用内存
    │       ├── memory/                # 内存归档
    │       │   └── YYYY-MM-DD-HHmm.md
    │       └── runs/                  # 运行会话
    │           └── {runId}.jsonl
    │
    └── mailboxes/                     # 空间邮箱
    └── taskboard.json                 # 空间任务板
```

                                      # (通过 CLAUDE_CONFIG_DIR 环境变量指定)

## 2. SQLite 数据库

### 2.1 数据库管理

**数据库路径**: `~/.aico-bot/aico-bot.db`

**连接管理**: `DatabaseManager` 单例，按路径缓存连接 (`Map<string, Database>`)

**PRAGMA 设置** (每次打开时应用):

| PRAGMA | 值 | 说明 |
|--------|-----|------|
| `journal_mode` | `WAL` | 预写日志模式，支持并发读写 |
| `synchronous` | `NORMAL` | 平衡性能与安全性 |
| `foreign_keys` | `ON` | 启用外键约束 |
| `busy_timeout` | `5000` | 忙等待超时 5 秒 |

**健康检查**: 连接复用时执行 `SELECT 1 FROM sqlite_master` 验证完整性。

**损坏恢复**: 检测到损坏模式 (`SQLITE_CORRUPT`, `SQLITE_NOTADB`, "database disk image is malformed") 时：
1. 将损坏的数据库文件 + WAL/SHM 文件重命名为 `.corrupt.{timestamp}`
2. 创建新数据库

### 2.2 迁移系统

**迁移元数据表**:

```sql
CREATE TABLE IF NOT EXISTS _migrations (
    namespace TEXT PRIMARY KEY,
    version INTEGER,
    applied_at INTEGER
);
```

| 字段 | 说明 |
|------|------|
| `namespace` | 模块命名空间 (如 `'app_manager'`, `'app_runtime'`, `'scheduler'`) |
| `version` | 当前迁移版本 |
| `applied_at` | 应用时间戳 (Unix ms) |

**迁移执行流程**:
1. 验证迁移列表 (版本号必须为正整数、无重复)
2. 按版本号升序排序
3. 创建 `_migrations` 表 (如不存在)
4. 获取当前命名空间版本 (默认 0)
5. 过滤出待执行迁移 (version > current)
6. 在单个事务中执行所有待迁移
7. 更新 `_migrations` 表版本

**注意**: 允许版本号序列中存在间隔 (仅警告不报错)。

## 3. 数据库表结构

### 3.1 调度任务表 (`scheduler_jobs`)

命名空间: `'scheduler'`

```sql
CREATE TABLE scheduler_jobs (
    id              TEXT PRIMARY KEY,     -- UUID v4
    name            TEXT NOT NULL,        -- 任务名称
    schedule_json   TEXT NOT NULL,        -- JSON: { kind, every/cron/once, timezone }
    enabled         INTEGER NOT NULL,     -- 0/1
    anchor_ms       INTEGER,              -- 锚点时间戳
    next_run_at_ms  INTEGER,              -- 下次执行时间
    last_run_at_ms  INTEGER,              -- 上次执行时间
    running_at_ms   INTEGER,              -- 当前运行开始时间
    consecutive_errors INTEGER DEFAULT 0, -- 连续错误次数
    status          TEXT DEFAULT 'idle',  -- idle/running/paused/disabled
    metadata_json   TEXT,                 -- 不透明元数据
    created_at      INTEGER NOT NULL,
    updated_at      INTEGER NOT NULL
);
```

### 3.2 调度运行日志表 (`scheduler_run_log`)

```sql
CREATE TABLE scheduler_run_log (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    job_id          TEXT NOT NULL REFERENCES scheduler_jobs(id) ON DELETE CASCADE,
    started_at      INTEGER NOT NULL,
    finished_at     INTEGER NOT NULL,
    duration_ms     INTEGER NOT NULL,
    outcome         TEXT NOT NULL,        -- useful/noop/error/skipped
    error           TEXT,
    metadata_json   TEXT
);

CREATE INDEX idx_run_log_job_started ON scheduler_run_log(job_id, started_at DESC);
```

### 3.3 已安装应用表 (`installed_apps`)

命名空间: `'app_manager'`

```sql
CREATE TABLE installed_apps (
    id                      TEXT PRIMARY KEY,       -- UUID v4
    spec_id                 TEXT NOT NULL,          -- 规范 ID
    space_id                TEXT NOT NULL,          -- 空间 ID
    spec_json               TEXT NOT NULL,          -- 安装时的规范快照
    status                  TEXT NOT NULL,          -- 应用状态
    pending_escalation_id   TEXT,                   -- 待处理升级 ID
    user_config_json        TEXT,                   -- 用户配置
    user_overrides_json     TEXT,                   -- 用户覆盖 (频率、通知级别、模型)
    permissions_json        TEXT,                   -- 权限 (granted/denied)
    installed_at            INTEGER NOT NULL,       -- 安装时间
    last_run_at             INTEGER,                -- 上次运行时间
    last_run_outcome        TEXT,                   -- 上次运行结果
    error_message           TEXT,                   -- 错误消息
    uninstalled_at          INTEGER,                -- 卸载时间 (v2 新增)

    UNIQUE(spec_id, space_id)
);

CREATE INDEX idx_installed_apps_space ON installed_apps(space_id);
CREATE INDEX idx_installed_apps_status ON installed_apps(status);
```

**迁移历史**:
- **v1**: 创建表 (不含 `uninstalled_at`)
- **v2**: 添加 `uninstalled_at INTEGER` 列

### 3.4 自动化运行表 (`automation_runs`)

命名空间: `'app_runtime'`

```sql
CREATE TABLE automation_runs (
    run_id          TEXT PRIMARY KEY,           -- UUID v4
    app_id          TEXT NOT NULL REFERENCES installed_apps(id) ON DELETE CASCADE,
    session_key     TEXT NOT NULL,              -- 会话键 (app-run-{runId[0:8]})
    status          TEXT NOT NULL,              -- running/ok/error/skipped/waiting_user
    trigger_type    TEXT NOT NULL,              -- schedule/event/manual/escalation_followup
    trigger_data    TEXT,                       -- 触发数据 JSON
    started_at      INTEGER NOT NULL,
    finished_at     INTEGER,
    duration_ms     INTEGER,
    tokens_used     INTEGER,
    error_message   TEXT
);

CREATE INDEX idx_runs_app_started ON automation_runs(app_id, started_at DESC);
CREATE INDEX idx_runs_status ON automation_runs(status);  -- v2 新增
```

### 3.5 活动记录表 (`activity_entries`)

```sql
CREATE TABLE activity_entries (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    app_id          TEXT NOT NULL REFERENCES installed_apps(id) ON DELETE CASCADE,
    run_id          TEXT REFERENCES automation_runs(run_id) ON DELETE CASCADE,
    session_key     TEXT,
    type            TEXT NOT NULL,              -- run_complete/run_skipped/run_error/milestone/escalation/output
    ts              INTEGER NOT NULL,           -- 时间戳
    content_json    TEXT NOT NULL,              -- 内容 JSON (summary, status, duration, error, etc.)
    user_response_json TEXT                     -- 用户响应 (升级场景)
);

CREATE INDEX idx_entries_app_ts ON activity_entries(app_id, ts DESC);
CREATE INDEX idx_entries_run ON activity_entries(run_id);  -- v2 新增
```

**迁移历史**:
- **v1**: 创建 `automation_runs` 和 `activity_entries` 表
- **v2**: 添加索引 `idx_entries_run` 和 `idx_runs_status`

**数据生命周期**: `pruneOldData()` 默认保留 **1 年** 的已完成运行记录。

## 4. 文件存储

### 4.1 对话持久化

**路径**: `{spacePath}/.aico-bot/conversations/{conversationId}.jsonl`

**格式**: JSON Lines，每行一个 SDK 事件记录。

### 4.2 应用运行会话

**路径**: `{spacePath}/.aico-bot/apps/{appId}/runs/{runId}.jsonl`

**写入**: `openSessionWriter()` 创建 SessionWriter，提供 `writeEvent(event)` 和 `writeTrigger(content)` 方法。

**读取**: `readSessionMessages()` 读取 JSONL 并转换为渲染进程兼容的 `Message[]` 格式。

**转换策略**: 累积-刷新 (accumulate-and-flush)
1. 累积连续 assistant 事件中的 thinking/tool_use 块到共享 `thoughts[]` 数组
2. tool-result user 事件合并到对应的 tool_use 思考中
3. 当 assistant 产生文本输出时刷新

### 4.3 内存系统文件

#### 用户级内存

| 文件 | 路径 | 说明 |
|------|------|------|
| 主文件 | `~/.aico-bot/user-memory.md` | 当前用户内存 |
| 归档目录 | `~/.aico-bot/user-memory/` | 归档文件 |

#### 空间级内存

| 文件 | 路径 | 说明 |
|------|------|------|
| 主文件 | `{spacePath}/.aico-bot/memory.md` | 当前空间内存 |
| 归档目录 | `{spacePath}/.aico-bot/memory/` | 归档文件 |

#### 应用级内存

| 文件 | 路径 | 说明 |
|------|------|------|
| 主文件 | `{spacePath}/.aico-bot/apps/{appId}/memory.md` | 当前应用内存 |
| 归档目录 | `{spacePath}/.aico-bot/apps/{appId}/memory/` | 归档文件 |
| 运行摘要 | `{spacePath}/.aico-bot/apps/{appId}/memory/run/` | 运行摘要文件 |

#### 内存文件结构

```markdown
# now

当前工作状态、活跃任务、重要上下文...

<!-- 2024-01-15T10:30:00.000Z by automation-run-xxx -->
## 2024-01-15-1030

具体执行记录和历史信息...
```

**压缩阈值**: 100KB (`COMPACTION_THRESHOLD_BYTES`)

**归档命名**: `YYYY-MM-DD-HHmm.md`

#### 写入操作

- **追加写入**: `appendToMemoryFile()` - 在内容前添加 `<!-- ISO-timestamp by source -->` 元数据注释
- **替换写入**: `replaceMemoryFile()` - 原子写入 (临时文件 + rename)

### 4.4 邮箱消息文件

**路径**: `~/.aico-bot/spaces/{spaceId}/mailboxes/{mailboxId}.jsonl`

**格式**: JSON Lines，每条消息包含:
- `index`: 递增索引
- `type`: 消息类型 (chat/direct/task_assignment/shutdown_request/permission_response)
- `from`: 发送者
- `to`: 接收者
- `content`: 消息内容
- `timestamp`: 时间戳

**原子写入**: 临时文件 + rename

**游标轮询**: 每个消费者维护 `lastReadIndex`，只读取新消息。

### 4.5 任务看板文件

**路径**: `~/.aico-bot/spaces/{spaceId}/taskboard.json`

**格式**: 单 JSON 文件，包含任务数组。

**任务结构**:
```json
{
  "id": "uuid",
  "title": "任务标题",
  "description": "任务描述",
  "status": "posted|claimed|in_progress|completed|failed",
  "assignedTo": "worker-id",
  "createdAt": 1234567890,
  "updatedAt": 1234567890
}
```

### 4.6 应用工作目录

**路径**: `{spacePath}/.aico-bot/apps/{appId}/`

**结构**:
```
{appId}/
├── memory.md              # 应用内存
├── memory/                # 内存归档
│   ├── YYYY-MM-DD-HHmm.md
│   └── run/
│       └── YYYY-MM-DD-HHmm.md
├── runs/                  # 运行会话
│   └── {runId}.jsonl
└── ...
```

安装时创建，卸载时删除 (`rmSync` 递归)。
