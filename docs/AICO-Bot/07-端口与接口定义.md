## 1. HTTP 服务器

### 1.1 服务器配置

| 参数 | 值 |
|------|-----|
| 默认端口 | **3847** |
| 监听地址 | `0.0.0.0` (所有接口) |
| 端口冲突策略 | 向上扫描最多 20 个端口 |
| 开发模式 | Vite 开发服务器运行在 `http://localhost:5173`，反向代理非 API 请求 |

### 1.2 认证机制

| 项目 | 说明 |
|------|------|
| Token 类型 | 内存字符串 (默认 6 位数字 PIN) |
| 自定义 Token | 4-32 位字母数字 |
| 传递方式 | `Authorization: Bearer <token>` 头部 或 `?token=` 查询参数 |

**认证豁免路由**:
- `POST /api/remote/login`
- `GET /api/remote/status`
- 静态资源文件
- Vite 开发服务器请求

## 2. WebSocket 端点

### 2.1 本地 WebSocket 服务器

| 参数 | 值 |
|------|-----|
| 路径 | `/ws` |
| 服务器 | `ws` 库 (WebSocketServer) |

### 2.2 认证流程

```
客户端连接
    │
    ▼
发送 { type: 'auth', payload: { token } }
    │
    ├──► 认证成功 → { type: 'auth:success' }
    │
    └──► 认证失败 → { type: 'auth:failed' } → 100ms 后关闭连接
```

### 2.3 客户端消息

| 类型 | Payload | 说明 |
|------|---------|------|
| `auth` | `{ token }` | 认证 |
| `subscribe` | `{ conversationId }` | 订阅对话 (需已认证) |
| `unsubscribe` | `{ conversationId }` | 取消订阅 |
| `ping` | - | 心跳 |

### 2.4 服务端广播

```typescript
broadcastToWebSocket(channel: string, data: any)
```

- 按 `conversationId` 路由事件到已订阅客户端
- 无订阅者时缓冲事件 (每 conversationId 最多 200 条，60 秒 TTL)
- 客户端订阅后立即刷新缓冲事件

```typescript
broadcastToAll(channel: string, data: any)
```

- 发送给所有已认证客户端，不论订阅状态

### 2.5 事件缓冲策略

| 参数 | 值 |
|------|-----|
| 每 conversationId 最大缓冲事件 | 200 |
| 缓冲 TTL | 60 秒 |
| 刷新时机 | 客户端订阅时 |
