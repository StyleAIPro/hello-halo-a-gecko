# 06 - 远程代理架构设计

## 1. 概述

AICO-Bot 支持将 AI 代理运行在远程服务器上，本地桌面应用通过 WebSocket + SSH 隧道与远程代理通信。这种架构实现了计算资源隔离、多客户端共享、以及利用远程服务器更强的计算能力。

### 1.1 架构组件

```
┌──────────────────────┐         ┌───────────────────────┐         ┌─────────────────────┐
│  AICO-Bot Desktop    │         │  SSH Tunnel (可选)     │         │  Remote Agent Proxy  │
│                      │         │                       │         │  (Node.js Server)    │
│  RemoteWsClient ─────┼────────►│  Local Port Forward   ├────────►│                      │
│  MCP Bridge          │  WS     │  (ssh2 库)            │  WS     │  Claude Agent SDK    │
│  Terminal Gateway    │         │                       │         │  MCP Servers         │
└──────────────────────┘         └───────────────────────┘         └─────────────────────┘
```

### 1.2 关键模块

| 模块 | 路径 | 说明 |
|------|------|------|
| RemoteWsClient | `src/main/services/remote-ws/remote-ws-client.ts` | WebSocket 客户端 |
| MCP Bridge | `src/main/services/remote-ws/aico-bot-mcp-bridge.ts` | 本地 MCP 工具桥接 |
| SshTunnelService | `src/main/services/remote-ssh/ssh-tunnel.service.ts` | SSH 隧道服务 |
| SSHManager | `src/main/services/remote-ssh/ssh-manager.ts` | 通用 SSH 连接管理 |
| Remote Agent Proxy | `packages/remote-agent-proxy/` | 独立 Node.js 代理服务器 |

## 2. Remote WebSocket 客户端

### 2.1 连接配置

| 参数 | 值 |
|------|-----|
| WebSocket 路径 (直连) | `ws://{host}:{port}/agent` |
| WebSocket 路径 (SSH 隧道) | `ws://localhost:{localTunnelPort}/agent` |
| 认证方式 | `Authorization: Bearer {token}` 头部 + `{ type: 'auth' }` 消息 |
| 连接超时 | **30 秒** |
| 认证超时 | **10 秒** |
| Ping 间隔 | **30 秒** |
| Pong 超时 | **90 秒** (超时则断开，代码 4001) |
| 重连策略 | 指数退避，起始 3000ms，最大 5 次 |

### 2.2 连接池

连接按 `serverId` 进行池化：

- 每个 serverId 维护一个连接
- 超过 30 分钟的连接会被回收
- 引用计数追踪每个连接的使用者

### 2.5 流式聊天

`sendChatWithStream()` 实现:

1. 发送 `{ type: 'claude:chat', sessionId, payload: { messages, options: { stream: true } } }`
2. 监听 `claude:stream`, `claude:usage`, `claude:complete`, `claude:error` 事件
3. 默认空闲超时: **30 分钟** (可配置)
4. 活动 (流、思考、终端输出) 重置空闲计时器
5. 活跃会话追踪在 `activeStreamSessions` Map 中

### 2.6 中断机制

`interrupt(sessionId)` 流程:

1. 发送 `{ type: 'claude:interrupt', sessionId }` 和 `{ type: 'close:session', sessionId }`
2. 等待 300ms 让排队事件处理完成
3. 设置 `isInterrupted` 标志阻止后续流事件
4. 拒绝所有未完成的流 Promise
5. 断开连接

## 3. MCP 桥接

### 3.1 概述

MCP Bridge 将本地 MCP 工具定义序列化后发送给远程代理。远程代理需要调用这些工具时，通过 WebSocket 发回请求，本地执行后返回结果。

### 3.2 桥接的工具

| 工具集 | 工具数量 | 说明 |
|--------|----------|------|
| ai-browser | 26 | 浏览器自动化 |
| gh-search | 8 | GitHub 搜索 |

**不桥接的工具**: aico-bot-apps, hyper-space 等业务逻辑工具 — 远程代理有各自的实现。

### 3.3 桥接协议

```
1. 认证后，客户端发送:
   { type: 'mcp:tools:register', payload: { tools, capabilities } }

2. 远程代理需要调用工具时发送:
   { type: 'mcp:tool:call', data: { toolName, args, callId } }

3. 本地执行后返回:
   { type: 'mcp:tool:response', payload: { callId, toolResult } }
```

## 4. SSH 隧道

### 4.1 SshTunnelService

#### 本地端口转发 (establishTunnel)

| 参数 | 说明 |
|------|------|
| 本地端口 | 动态分配，从 remotePort 开始 (默认 8080)，最多尝试 100 次 |
| SSH 就绪超时 | 30000ms |
| Keepalive 间隔 | 30000ms |
| Keepalive 最大次数 | 3 |
| 认证方式 | 密码或私钥 |

**隧道共享**: 多个空间可以共享同一个隧道 (通过 `spaces` Set 引用计数)。只有最后一个空间断开时隧道才关闭。

**数据流向**:
```
本地应用 → 本地 TCP 端口 → SSH 连接 → client.forwardOut() → 远程服务器 → 目标端口
```

#### 反向隧道 (createReverseTunnel)

复用现有 SSH 连接，通过 `client.forwardIn('127.0.0.1', remoteListenPort)` 使远程端监听端口，incoming 连接转发到本地目标端口。

### 4.2 SSHManager

通用 SSH 连接处理器，提供:

| 方法 | 说明 |
|------|------|
| `connect()` / `disconnect()` | 连接生命周期 |
| `executeCommand()` | 远程命令执行 |
| `executeCommandFull()` | 完整命令执行 (含 stdin) |
| `executeCommandStreaming()` | 流式命令执行 |
| `uploadFile()` / `downloadFile()` / `mkdir()` | SFTP 文件操作 |
| `createLocalPortForward()` | `-L` 风格端口转发 |
| `createReversePortForward()` | `-R` 风格端口转发 |
| `executeShell()` / `writeShell()` | 交互式 Shell |
| `ensureConnected()` | 健康检查 + 重连 |

### 4.3 端口管理 (Windows 感知)

| 平台 | 端口检查 | 进程终止 |
|------|----------|----------|
| Windows | `netstat -ano \| findstr` | `taskkill /F /PID` |
| macOS/Linux | `lsof -i :port -t` | `kill` |

## 5. 远程服务器管理

### 5.1 远程服务器配置

每个远程服务器包含以下配置:

- `id`: 服务器唯一标识
- `name`: 显示名称
- `host`: 主机地址
- `port`: SSH 端口 (默认 22)
- `authType`: `password` 或 `privateKey`
- `username`: SSH 用户名
- `password` / `privateKeyPath`: 认证凭据
- `useSshTunnel`: 是否使用 SSH 隧道
- `agentPath`: 远程代理路径

### 5.2 远程代理部署

部署流程:
1. SSH 连接目标服务器
2. 上传代理代码 (remote-agent-proxy 包)
3. 安装依赖 (`npm install --production`)
4. 配置环境变量
5. 启动代理进程

### 5.3 远程代理生命周期管理

| 操作 | 说明 |
|------|------|
| 启动代理 | SSH 执行启动脚本 |
| 停止代理 | SSH 发送终止信号 |
| 检查状态 | SSH 查询进程状态 |
| 查看日志 | SSH 读取日志文件 |
| 更新代理 | 上传新版本 + 重启 |

## 6. 数据流详解

### 6.1 远程对话完整数据流

```
用户输入消息
    │
    ▼
Renderer: api.agent.sendMessage()
    │ (IPC)
    ▼
Main: send-message.ts
    │
    ▼ (space.claudeSource === 'remote')
executeRemoteMessage()
    │
    ├──► SshTunnelService.establishTunnel() (如启用)
    │     SSH 连接 → 本地端口转发 → ws://localhost:LOCAL_PORT
    │
    ├──► RemoteWsClient.connect()
    │     ws://localhost:LOCAL_PORT/agent (隧道模式)
    │     或 ws://remote-host:PORT/agent (直连模式)
    │
    ├──► MCP Bridge 注册
    │     收集本地 MCP 工具 (ai-browser, gh-search)
    │     发送到远程代理
    │
    ├──► RemoteWsClient.sendChatWithStream()
    │     发送 claude:chat 消息
    │     监听 claude:stream 事件
    │
    ▼
远程代理 (Claude Agent SDK)
    │
    ├──► 处理消息
    ├──► 需要本地工具时 → mcp:tool:call → 本地执行 → mcp:tool:response
    ├──► 流式返回结果 → claude:stream
    └──► 完成 → claude:complete
    │
    ▼
RemoteWsClient 接收事件
    │
    ├──► 转发到 send-message.ts
    ├──► 持久化对话消息
    └──► 发送事件到 Renderer
              agent:thought
              agent:thought-delta
              agent:message
              agent:tool-call
              agent:tool-result
              agent:complete
```

### 6.2 远程代理思考过程数据流

```
远程服务器 (claude-manager.ts)     本地客户端 (send-message.ts)    前端 (chat.store.ts)
─────────────────────────────────  ────────────────────────────    ────────────────────────
SDK stream_event                   RemoteWsClient events            Zustand store updates
├─ content_block_start (thinking) ─► 'thought' event ────────────► handleAgentThought()
├─ content_block_delta             ──► 'thought:delta' event ──────► handleAgentThoughtDelta()
└─ content_block_stop              ──► (complete signal)           ──► thought.isStreaming = false
```

**调试要点**:
- `hasStreamEvent` 标志控制回退行为
- `sessionId` 路由匹配 `conversationId`
- `streamingBlocks` Map 追踪活跃块 (按索引)
- 回退路径: 当无 stream_event 时，从 assistant 消息处理思考块

## 7. 连接池设计

### 7.1 连接管理

```typescript
interface PooledConnection {
  serverId: string
  client: RemoteWsClient
  refCount: number
  createdAt: number
  lastUsedAt: number
}
```

### 7.2 连接生命周期

```
getConnection(serverId)
    │
    ├──► 池中已有且未过期 → refCount++, 返回
    │
    ├──► 池中有但已过期 (30 分钟) → 关闭旧连接, 创建新连接
    │
    └──► 池中无 → 创建新连接, 加入池
```

### 7.3 心跳机制

```
每 30 秒: 发送 ping
    │
    ├──► 收到 pong → 重置 pongTimeout (90 秒)
    │
    └──► 90 秒未收到 pong → 断开连接 (code 4001) → 触发重连
```
