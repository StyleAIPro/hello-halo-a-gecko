  ⚠️  现有远程方案的核心问题

  问题 1：远程 Agent Proxy 过于简化

  当前的 packages/remote-agent-proxy/src/claude-manager.ts 只是一个简单的包装器：

  // 远程服务器上的 ClaudeManager
  async streamChat(sessionId, messages, options, onToolCall, onTerminalOutput) {
    const session = this.getSession(sessionId)  // 创建本地 V2 Session
    await session.send(messageContent)
    for await (const event of session.stream()) {
      // 这里处理 tool_call, terminal, text 等事件
      // 但是：没有 MCP 服务器、没有 AI Browser、没有 Halo Apps
    }
  }

  问题: 远程服务器上运行的是"简化版"Claude Code，缺少所有扩展功能。

  问题 2：消息历史未正确传递

  // executeRemoteMessage 中的消息构建
  const messageArray = [
    { role: 'user', content: message }  // ❌ 只有当前消息，没有历史！
  ]

  问题: 多轮对话需要完整的消息历史，否则 Claude 不记得之前的对话。

  问题 3：会话状态未同步

  - 本地 v2Sessions Map 保存活跃会话
  - 远程也有自己的 sessions Map
  - 两边的会话状态完全独立，导致：
    - 本地停止生成，远程继续
    - 本地会话关闭，远程会话泄漏

  问题 4：工具审批流程缺失

  本地流程：
  agent:tool-call → 前端显示审批对话框 → 用户选择 → agent:approve/reject-tool → SDK

  远程流程：
  agent:tool-call → 前端显示审批对话框 → 用户选择 → ❌ 未发送到远程
