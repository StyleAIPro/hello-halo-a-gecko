# Feature: Add Server 部署进度内嵌到服务器卡片

## 元信息

- **时间**: 2026-04-22
- **状态**: draft
- **优先级**: P1
- **指令人**: StyleAIPro
- **影响范围**: 仅前端（renderer）
- **PRD 级别**: feature

## 需求背景

当前 Add Server 时，部署进度（进度条 + 状态消息）显示在一个独立的模态弹窗中，终端命令行输出显示在服务器卡片的展开区域。问题：

1. **弹窗遮挡卡片**：用户无法同时看到进度条和终端输出
2. **切换成本高**：需要关闭弹窗才能看到命令行详情
3. **操作不连贯**：添加服务器 → 弹窗 → 关闭 → 找到卡片展开 → 才能看到输出

## 需求描述

将 Add Server 的部署进度直接内嵌到服务器卡片中，去掉独立的进度弹窗。用户点击"添加服务器"并提交后：

1. 服务器列表中立即出现新卡片（或使用内联表单）
2. 进度条直接显示在卡片头部
3. 终端输出自动展开显示在卡片下方
4. 完成/失败后卡片恢复正常状态

## 技术方案

### 现状分析

```
当前流程:
  点击添加 → 模态弹窗(表单) → 提交 → 模态弹窗(进度条) → 完成 → 关闭弹窗
                                                    ↕ 并行
                                              服务器卡片(终端输出, 需手动展开)

目标流程:
  点击添加 → 模态弹窗(表单) → 提交 → 弹窗关闭 → 服务器卡片(进度条 + 终端输出自动展开)
```

### UI 改动

**RemoteServersSection.tsx 修改：**

1. **服务器卡片增加进度条区域**（卡片头部下方，终端输出上方）：
   - 当 `addProgress.serverId === server.id` 且部署进行中时显示
   - 包含：进度条 + 百分比 + 状态消息 + spinner/check/error 图标
   - 布局与当前弹窗中的进度条一致，但内嵌在卡片内

2. **提交后自动关闭表单弹窗**：
   - `handleAddServer()` 提交后立即 `setShowAddDialog(false)`
   - 不再在弹窗中显示进度

3. **卡片自动展开**：
   - 部署期间卡片自动展开（已有逻辑：第 322 行 `setExpandedServers`）
   - 保持不变

4. **去掉弹窗中的进度视图**：
   - 删除第 1263-1337 行的 `saving && !editingServer && addProgress` 分支
   - 弹窗仅用于表单输入

5. **完成/失败处理**：
   - 完成/失败后进度条区域消失（`addProgress` 被清理）
   - 卡片保持展开，用户可查看终端输出的完整日志
   - 通过已有的 status change 事件刷新服务器状态

### 数据流（不变）

```
后端 emitDeployProgress → IPC/HTTP → handleDeployProgress → addProgress state + terminalEntries
```

数据流完全不变，只是渲染位置从弹窗移到卡片内。
